# Инструкции для агента: Создание Block Blast Core + Редактор уровней

В данном файле собраны технические требования, архитектура и этапы разработки playable core-игры в стиле Block Blast с редактором уровней.

## Общий контекст проекта

Разработка ведется под мобильный WebView (vertical mobile-first, 9:16).
Необходимо реализовать только **core-игру** и **редактор уровней**. Мета-механики (дерево, магазин, задания и т.д.) реализовывать НЕ нужно.

### Технологический стек
* **Next.js** (последняя версия, App Router)
* **React** (последняя версия)
* **TypeScript**
* **Pixi.js** (версия 8+) — для рендеринга игры
* **Zustand** (версия 5+) — для стейт-менеджмента
* **Sass (SCSS modules)** — для стилизации
* **Howler** — для звуков (опционально)

### Основной роутинг
* `/editor` — страница с редактором уровня и live preview игры.

## Описание core-игры и механик

### 1. Поле
* Сетка `8x8` (размер задается в `LevelConfig.grid`).
* Клетка поля:
  ```ts
  type BoardCell = {
    id: string;
    filled: boolean;
    color?: string;
    figureId?: string;
    hasWater?: boolean;
  };
  ```

### 2. Фигуры
* 15 базовых фигур (`FigureShape`), которые нельзя вращать.
* Фигуры размещаются игроком вручную из 3 нижних слотов.
* Генерация по весам (`DEFAULT_FIGURE_WEIGHTS`).

### 3. Очки и вода
* Начисляются **только** при очистке линий или при использовании бустера (за установку фигуры очки не начисляются).
* `1 клетка = 1 очко (или вода)`.
* При одновременной очистке нескольких линий применяется множитель (`total = baseScore * clearedLinesCount * boosterMultiplier`).

### 4. Победа и поражение
* **Победа:** `score >= targetScore`. Показывается victory overlay.
* **Поражение:** Ни одну из 3 фигур нельзя разместить.
  * Сначала показывается "Защита от поражения" (предложение очистить поле за стоимость в воде/очках или бесплатно в зависимости от настроек).
  * Если игрок соглашается — поле очищается, игра продолжается.
  * Если отказывается — поражение (lost overlay).

### 5. Бустеры
1. **Собрать всё (Collect All):** Очищает все клетки поля и начисляет за них очки.
2. **Множитель (Multiplier):** Действует до конца уровня. Удваивает все очки (x2).
3. **Молоток (Hammer):** Выбор области 4x4, удаление всех занятых блоков в ней с начислением очков.

## Пошаговые Промпты для реализации

### Промпт 1. Подготовка проекта и архитектуры
Ты senior frontend/game developer. Нужно создать core-игру Block Blast-like в Next.js + React + TypeScript.
На этом этапе не реализуй полную игру. Подготовь архитектуру проекта, типы, конфиги и страницу `/editor`.

**Требования:**
1. Создай страницу `src/app/editor/page.tsx`.
2. Создай структуру папок (`entities/game`, `widgets/game-core`, `widgets/level-editor`, `shared/lib`).
3. Добавь типы для `LevelConfig`, `BoardCell`, `FigureShape`, `FigureInstance`, `GameState`, `BoosterType`.
4. Добавь 15 базовых фигур и `DEFAULT_FIGURE_WEIGHTS`.
5. Добавь `DEFAULT_LEVELS` с 2-3 тестовыми уровнями.
6. Добавь функции `normalizeLevelConfig` и `validateLevelConfig`.
7. Сделай базовый layout `/editor`: слева панель редактора, справа mobile preview 9:16 (пока с placeholder-компонентом).
8. Используй TypeScript и SCSS modules.
9. Код должен быть готов к следующему этапу (игровой логике).

### Промпт 2. Чистая игровая логика
Реализуй чистую игровую логику без Pixi-анимаций.
* Функции: `createEmptyBoard`, `canPlaceFigure`, `placeFigure`, `findCompletedLines`, `clearCompletedLines`, `calculateScore`, `generateFigureSet`, `canPlaceAnyFigure`, `checkWinCondition`, `checkLoseCondition`.
* Логика доски должна брать размеры из конфига. Очки начисляются только за линии/бустеры с учетом multiplier.

### Промпт 3. Базовый Pixi game preview
Реализуй базовый `GameCore` на Pixi.js (версия 8+).
* React-компонент `GameCore`, инициализирующий `Application`.
* Отрисовка: фон, сетка, progress score 0/targetScore, 3 фигуры в слотах.
* Псевдо-3D отрисовка кубиков через `Graphics`. При изменении конфига игра пересоздается. Без drag-and-drop.

### Промпт 4. Drag-and-drop фигур
* Фигуры в слотах меньше и левитируют.
* При pointerdown фигура увеличивается и следует за pointer.
* При перемещении над полем подсвечиваются клетки placement (зеленым/красным/без подсветки).
* Pointerup: успешная установка (bounce-анимация, проверка линий) или возврат в слот.
* При размещении всех 3 фигур — генерация нового набора.

### Промпт 5. Очистка линий, scoring и победа
* Очистка собранных горизонтальных и вертикальных линий.
* Анимация исчезновения блоков (scale/fade/pop) и полет частиц воды в progress bar.
* Реализация комбо-множителей (x2, x3) и бустера Multiplier.
* Экран победы (victory overlay) с кнопкой рестарта.

### Промпт 6. Поражение и защита от поражения
* Проверка возможности хода после каждого placement.
* Если ходов нет — показ overlay "Защита от поражения" (кнопка очистки поля или завершения игры).
* При очистке — продолжение игры. При завершении — lose overlay.

### Промпт 7. Бустеры
* Реализация 3 бустеров: Collect All, Multiplier, Hammer.
* Включение режима Hammer (выбор области 4x4, перетаскивание рамки, подтверждение/отмена).
* Кнопки бустеров в HUD игры с отображением количества.

### Промпт 8. Редактор уровней
* Левая панель: форма параметров (levelId, targetScore, фигуры, бустеры, защита, visual).
* Импорт/экспорт JSON, копирование в буфер, валидация.
* Применение изменений (кнопка "Применить") обновляет preview справа.

### Промпт 9. Initial board editor
* Добавление интерактивной сетки в форму редактора для настройки `initialBoard`.
* Возможность кликом включать/выключать блоки и выбирать их цвет.
* Синхронизация с JSON-конфигом и сброс preview при применении.

### Промпт 10. Визуальная полировка
* Деревянный фон, тени под фигурами, idle-анимация левитации.
* Улучшение частиц воды, эффектов бустеров.
* Звуки через Howler (pick, place, invalid, clear, booster, win, lose), отключаемые в настройках.

### Промпт 11. Финальная проверка и edge cases
* Проверка всех edge cases, адаптивности, работы на тач-экранах и сборка Next.js проекта (`npm run build`).
